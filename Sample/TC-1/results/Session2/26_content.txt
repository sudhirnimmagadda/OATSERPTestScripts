<H1 class=chapter><SPAN class=secnum>3</SPAN> <!--BOLOC E1548702-998750-4-->Oracle OpenScript <A id=sthref213 name=sthref213></A><A id=sthref214 name=sthref214></A>Tutorial<!--EOLOC E1548702-998750-4--></H1>
<P><!--BOLOC E1548702-998763-4-->This tutorial walks you through the main features of Oracle OpenScript. The tutorial consists of the following examples:<!--EOLOC E1548702-998763-4--></P>
<UL>
<LI>
<P><!--BOLOC E1548702-998778-4--><SPAN class=bold>Starting the Avitek Medical Records Sample Application</SPAN> - explains how to start the sample Avitek Medical Records Server for use with this tutorial.<!--EOLOC E1548702-998778-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-998795-4--><SPAN class=bold>Starting Oracle OpenScript</SPAN> - describes how to start OpenScript.<!--EOLOC E1548702-998795-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-998812-4--><SPAN class=bold>Creating a Web Functional Test Script</SPAN> - describes how to create a Web Functional test script project and record a script. <!--EOLOC E1548702-998812-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-998829-4--><SPAN class=bold>Working with Scripts</SPAN> - explains the features of the OpenScript script tree and how to examine the structure and content of a recorded script.<!--EOLOC E1548702-998829-4--></P></LI>
<LI>
<P><SPAN class=bold>Playing Back a Web Functional Test Script</SPAN> - explains the procedure for playing back Web Functional Test scripts that you have recorded and viewing the results.</P></LI>
<LI>
<P><!--BOLOC E1548702-998863-4--><SPAN class=bold>Adding Tests to the Script</SPAN> - explains how to add tests to your OpenScript scripts. <!--EOLOC E1548702-998863-4--></P></LI>
<LI>
<P><SPAN class=bold>Creating an HTTP Test Script</SPAN> - describe how to create an HTTP protocol load test script project and record a load test script.</P></LI>
<LI>
<P><SPAN class=bold>Creating an HTTP Test Script with Databanks</SPAN> - describe how to create an HTTP protocol load test script project and explains how to configure and use Databanks to run iterative tests on a login form using data from an external file.</P></LI>
<LI>
<P><!--BOLOC E1548702-998897-4--><SPAN class=bold>Stopping the Avitek Medical Records Server</SPAN> - explains how to stop the sample Avitek Medical Records Server used with this tutorial.<!--EOLOC E1548702-998897-4--></P></LI></UL>
<P><!--BOLOC E1548701-998893-1-->The tutorial is designed to be followed sequentially from beginning to end. Many of the examples are interrelated and build upon the steps in previous examples.<!--EOLOC E1548701-998893-1--></P>
<DIV class=sect1><!-- infolevel="all" infotype="General" --><A id=sthref215 name=sthref215></A>
<H2 id=autoId0 class=sect1><SPAN class=secnum>3.1</SPAN> <!--BOLOC E1548702-998920-4-->Starting the Avitek Medical Records Sample Application<A id=sthref216 name=sthref216></A><!--EOLOC E1548702-998920-4--></H2>
<P><!--BOLOC E1548702-998928-4-->The tutorial uses the Avitek Medical Records Sample Application to record and playback OpenScript scripts. The WebLogic server and Avitek Medical Records Sample Application need to be started on the system before creating OpenScript script projects and recording scripts.<!--EOLOC E1548702-998928-4--></P>
<P><!--BOLOC E1548702-998934-4-->To start the Weblogic Server and Avitek Medical Records Sample Application:<!--EOLOC E1548702-998934-4--></P>
<OL>
<LI>
<P>Select <SPAN class=bold>Programs</SPAN> from the <SPAN class=bold>Start</SPAN> menu and then select <SPAN class=bold>Sample</SPAN> from the <SPAN class=bold>Oracle Application Testing Suite</SPAN> menu.</P></LI>
<LI>
<P>Select <SPAN class=bold>Start MedRec Application</SPAN>.</P></LI>
<LI>
<P>Wait until the Avitek Medical Records Sample Application starts in the Weblogic server.</P></LI>
<LI>
<P>Close the browser window after the server and sample application are started. When recording OpenScript scripts, the browser window will automatically open again to specify the Web address to record.</P></LI></OL></DIV><!-- class="sect1" -->
<DIV class=sect1><!-- infolevel="all" infotype="General" --><A id=sthref217 name=sthref217></A>
<H2 id=autoId1 class=sect1><SPAN class=secnum>3.2</SPAN> <!--BOLOC E1548702-999057-4-->Starting Oracle OpenScript<!--EOLOC E1548702-999057-4--></H2>
<P><!--BOLOC E1548702-999062-4-->To start OpenScript:<!--EOLOC E1548702-999062-4--></P>
<UL>
<LI>
<P>Select <SPAN class=bold>Programs</SPAN> from the <SPAN class=bold>Start</SPAN> menu, then select <SPAN class=bold>OpenScript</SPAN> from the <SPAN class=bold>Oracle Application Testing Suite</SPAN> Start menu.</P></LI></UL></DIV><!-- class="sect1" -->
<DIV class=sect1><!-- infolevel="all" infotype="General" --><A id=sthref218 name=sthref218></A>
<H2 id=autoId2 class=sect1><SPAN class=secnum>3.3</SPAN> <!--BOLOC E1548702-999126-4-->Example 1: Creating a Web Functional Test Script<A id=sthref219 name=sthref219></A><A id=sthref220 name=sthref220></A><!--EOLOC E1548702-999126-4--></H2>
<P><!--BOLOC E1548702-999136-4-->This example illustrates the creation of an OpenScript Web Functional test script project and recording a script. A script project creates the basic structure of an OpenScript script. Initially, the script project includes only the Initialize, Run, and Finish script nodes with the underlying Java code. You use the recording capabilities of OpenScript to record page navigations and generate the Java code for the script.<!--EOLOC E1548702-999136-4--></P>
<DIV class=sect2><!-- infolevel="all" infotype="General" --><A id=sthref221 name=sthref221></A>
<H3 id=autoId3 class=sect2><SPAN class=secnum>3.3.1</SPAN> <!--BOLOC E1548702-999147-4-->Creating a Web Functional Test Script Project<!--EOLOC E1548702-999147-4--></H3>
<P><!--BOLOC E1548702-999152-4-->To create a Web Functional Test Script Project:<!--EOLOC E1548702-999152-4--></P>
<OL>
<LI>
<P><!--BOLOC E1548702-999167-4-->Select <SPAN class=bold>New</SPAN> from the <SPAN class=bold>File</SPAN> menu.<!--EOLOC E1548702-999167-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999196-4-->Select <SPAN class=bold>Web</SPAN> under the <SPAN class=bold>Functional Testing (Browser/GUI Automation)</SPAN> folder.<!--EOLOC E1548702-999196-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999227-4-->Click <SPAN class=bold>Next</SPAN>.<!--EOLOC E1548702-999227-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999246-4-->Make sure the Repository is set to Default.<!--EOLOC E1548702-999246-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999266-4-->Enter WebTutor as the script name.<!--EOLOC E1548702-999266-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999276-4-->Click <SPAN class=bold>Finish</SPAN>. OpenScript creates the script project and shows the Initialize, Run, and Finish nodes in the Script View as follows:<!--EOLOC E1548702-999276-4--></P>
<DIV class=figure>
<P class=titleinfigure><A id=sthref222 name=sthref222></A>Figure 3-1 <!--BOLOC E1548702-999295-4-->OpenScript Script Project Tree<!--EOLOC E1548702-999295-4--></P><IMG title="Description of Figure 3-1 follows" alt="Description of Figure 3-1 follows" src="img/os_scp1.gif" width=122 longDesc="img_text/os_scp1.htm" height=65><BR><A id=sthref223 href="img_text/os_scp1.htm" name=sthref223>Description of "Figure 3-1 OpenScript Script Project Tree"</A><BR><BR></DIV><!-- class="figure" --></LI></OL></DIV><!-- class="sect2" -->

<H3 id=autoId4 class=sect2><SPAN class=secnum>3.3.2</SPAN> <!--BOLOC E1548702-999339-4-->Recording a Web Functional Test Script<A id=sthref225 name=sthref225></A><!--EOLOC E1548702-999339-4--></H3>
<P><!--BOLOC E1548702-999347-4-->Recording scripts captures the page navigations and generates the Java code that will be used to drive script playback for testing purposes. Web Functional test scripts record and playback actions performed on browser objects. <!--EOLOC E1548702-999347-4--></P>
<P><!--BOLOC E1548702-999353-4-->In this example, we will open the Avitek Medical Records sample application and log in as a patient. During recording, we will add a Text Matching test to verify that the log in was successful. <!--EOLOC E1548702-999353-4--></P>
<P><!--BOLOC E1548702-999359-4-->To record a Web Functional Test Script:<!--EOLOC E1548702-999359-4--></P>
<OL>
<LI>
<P><!--BOLOC E1548702-999374-4-->Select <SPAN class=bold>Record</SPAN> from the <SPAN class=bold>Script</SPAN> menu or click the Record toolbar button. OpenScript opens a browser window and the OpenScript toolbar window.<!--EOLOC E1548702-999374-4--></P></LI>
<LI>
<P>Enter <CODE>http://&lt;</CODE><CODE><SPAN class=codeinlineitalic>machineName</SPAN></CODE><CODE>&gt;:7011/medrec/</CODE> into the browser Address line and press <SPAN class=bold>Enter</SPAN>.</P></LI>
<LI>
<P><!--BOLOC E1548702-999438-4-->When the application loads, click <SPAN class=bold>Start using MedRec!</SPAN>. A second browser window opens with Avitek Medical Records Sample Application.<!--EOLOC E1548702-999438-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999457-4-->Click Login under the Patient section.<!--EOLOC E1548702-999457-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999467-4-->Enter <CODE>fred@golf.com</CODE> as the Email address.<!--EOLOC E1548702-999467-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999488-4-->Enter <CODE>weblogic</CODE> as the password.<!--EOLOC E1548702-999488-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999507-4-->Click <SPAN class=bold>Submit</SPAN>.<!--EOLOC E1548702-999507-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999526-4-->Click the Text Matching Test button on the OpenScript floating toolbar.<!--EOLOC E1548702-999526-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999536-4-->Enter <CODE>WebText1</CODE> as the Test name.<!--EOLOC E1548702-999536-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999555-4-->Enter <CODE>Successfully logged in! Click here to continue</CODE> as the Text to Match.<!--EOLOC E1548702-999555-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999574-4-->Make sure <SPAN class=bold>Source</SPAN> is set to <SPAN class=bold>HTML Display Contents</SPAN>.<!--EOLOC E1548702-999574-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999593-4-->Make sure <SPAN class=bold>Pass when</SPAN> is set to <SPAN class=bold>Pass if present</SPAN>.<!--EOLOC E1548702-999593-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999612-4-->Make sure <SPAN class=bold>Match</SPAN> is set to <SPAN class=bold>Exact</SPAN>.<!--EOLOC E1548702-999612-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999631-4-->Make sure V<SPAN class=bold>erify only, never fail</SPAN> is selected.<!--EOLOC E1548702-999631-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999650-4-->Click <SPAN class=bold>OK</SPAN>.<!--EOLOC E1548702-999650-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999669-4-->Click <SPAN class=underline>Successfully logged in! Click here to continue</SPAN>.<!--EOLOC E1548702-999669-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999688-4-->Click Logout in the Avitek Medical Records application.<!--EOLOC E1548702-999688-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999698-4-->Close the Avitek Medical Records application browser window.<!--EOLOC E1548702-999698-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-999708-4-->Close the Weblogic Server Avitek Medical Records Sample Application browser window.<!--EOLOC E1548702-999708-4--></P>
<P><!--BOLOC E1548702-999713-4-->Recording automatically stops when you close the second browser window. The script tree view includes plus icons to indicate additional nodes have been added to the script as follows:<!--EOLOC E1548702-999713-4--></P>
<DIV class=figure>
<P class=titleinfigure><A id=CHDJBEEI name=CHDJBEEI></A>Figure 3-2 <!--BOLOC E1548702-999724-4-->OpenScript Script Tree After Recording<!--EOLOC E1548702-999724-4--></P><IMG title="Description of Figure 3-2 follows" alt="Description of Figure 3-2 follows" src="img/os_scp2.gif" width=127 longDesc="img_text/os_scp2.htm" height=71><BR><A id=sthref226 href="img_text/os_scp2.htm" name=sthref226>Description of "Figure 3-2 OpenScript Script Tree After Recording"</A><BR><BR></DIV><!-- class="figure" --></LI>

<H2 id=autoId24 class=sect1><SPAN class=secnum>3.9</SPAN> <!--BOLOC E1548702-1003337-4-->Stopping the Avitek Medical Records Server<A id=sthref316 name=sthref316></A><!--EOLOC E1548702-1003337-4--></H2>
<P><!--BOLOC E1548702-1003345-4-->When finished with the tutorial be sure to stop the sample Medical Records Server. If you want to try using scripts recorded against the sample Medical Records Server application in Oracle Load Testing or Oracle Test Manager tutorials, leave the Avitek Medical Records Server running until after completing those trials.<!--EOLOC E1548702-1003345-4--></P>
<P><!--BOLOC E1548702-1003351-4-->To stop the Weblogic Server and Avitek Medical Records Server:<!--EOLOC E1548702-1003351-4--></P>
<OL>
<LI>
<P>Select <SPAN class=bold>Programs</SPAN> from the <SPAN class=bold>Start</SPAN> menu and then select <SPAN class=bold>Sample</SPAN> from the <SPAN class=bold>Oracle Application Testing Suite</SPAN> menu.</P></LI>
<LI>
<P>Select <SPAN class=bold>Stop MedRec Application</SPAN>.</P></LI>
<LI>
<P><!--BOLOC E1548702-1003454-4-->Wait until the Weblogic server stops.<!--EOLOC E1548702-1003454-4--></P></LI>
<LI>
<P><!--BOLOC E1548702-1003464-4-->Close the command windows after the server stops. <!--EOLOC E1548702-1003464-4--></P></LI></OL></DIV><!-- class="sect1" --></DIV><!-- class="ind" -->
